## 项目概述

**敞亮伴侣 (OpenVision Companion)** — 基于 lark-cli 的桌面端 AI 入职导航 Agent，帮助敞亮科技新员工更快建立公司上下文。通过对话式 UI 提供覆盖飞书全数据源（Wiki、文档、表格、IM 消息、搜索）的 RAG 知识检索。

- **阶段**：提案期（2026 年 6 月）
- **提交截止**：2026-06-30（敞亮AI共创计划首届大赛，岗位优化创新类赛道）
- **作者**：fly（前端开发），raph 提供 lark-cli 咨询
- **时间目标**：4 周出可用原型

## 架构设计

```
桌面应用 (Electron + Vue 3)
├── 对话 UI（Pull 模式：用户主动提问）
├── Push 通知（系统通知栏）
├── Agent 核心
│   ├── 对话管理（Pull）
│   ├── 上下文调度（Push）
│   └── 知识检索层（RAG）
└── lark-cli 数据接入层
    └── 文档 | Wiki | IM | 表格 | 搜索
```

## 技术选型

| 层 | 方案 | 原因 |
|----|------|------|
| 桌面壳 | Electron + Vue 3 | 复用现有前端技术栈（customer-im-h5 也是 Vue 3） |
| 数据接入 | lark-cli CLI | 已有全权限飞书集成 |
| 知识索引 | 本地向量数据库（LanceDB） | 无需服务端，数据本地存储 |
| AI 对话 | Claude API（通过 lark-cli 调用） | 公司 AI LAB 推荐 |
| Push 通知 | Electron Notification API | 原生系统通知 |
| 备选壳 | Tauri | 更轻量但生态不如 Electron 成熟 |

## 两大知识域

- **公司通用域**（`knowledge/company/`）：企业文化、HR/行政/财务制度、薪酬福利、新人培训计划。来源：公司规章制度 Wiki、敞亮文化 Wiki、飞书表格。
- **部门业务域**（`knowledge/department/{dept}/`）：产品背景、技术架构、当前版本需求、团队分工。来源：部门 Wiki、IM 群聊、技术文档。

## 两种交互模式

- **Pull（主动查询）**：用户提问 → Agent 判断知识域 → 通过 lark-cli 搜索相关文档 → 获取内容 → 生成回答并附引用来源。
- **Push（主动推送）**：定时任务（每日早晨）→ 读取用户培训计划（表格）→ 对比当前进度 → 读取当前版本需求文档 → 检索相关群聊近期讨论 → 生成今日建议 → 系统通知推送。

## 关键设计决策

1. **仅桌面端，不做 Web**：lark-cli 是 CLI 工具，Web 应用无法调用。
2. **本地向量数据库（LanceDB）**：无后端服务，索引和检索全部本地运行，保障数据隐私，零运维成本。
3. **双重上下文架构**：公司通用和部门业务知识分层组织，用户可切换上下文域。
4. **AI 能力基线植入**：产品本身就是一个 L3 级 AI Agent，新人在使用过程中自然学会 AI 协作模式，而非通过培训灌输。
