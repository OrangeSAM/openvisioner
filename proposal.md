# 敞亮伴侣：基于飞书全数据的 AI 入职导航系统

## 一、你发现了什么问题或机会？

### 当前现状

敞亮科技在飞书上沉淀了大量组织知识——8 个 Wiki 空间、数十份文档、多个群聊——但新人入职时面临**严重的信息断层**：

| 现状 | 痛点 |
|------|------|
| 翻飞书 Wiki | 十几个空间，文档散落在不同角落，不知道哪些跟自己相关 |
| 问导师和同事 | 反复打扰，信息碎片化，老员工也未必知道全貌 |
| 权限申请 | 不知道要申请什么权限，申请了还要等 IT 审批，影响效率 |
| 业务理解 | 单篇文档只是切片，新人很难串起产品来龙去脉与演进方向 |
| 制度与福利 | HR、行政、财务制度散落在各处，入职培训之后容易遗忘 |

### 真实案例

fly（一笔），前端开发，6 月 4 日入职。为了理解智能客服产品，他需要：
- 翻遍 8 个 Wiki 空间
- 找到并阅读 10+ 篇相关文档
- 向 5+ 位同事请教
- 花 3 天时间才拼凑出完整的业务全景

### 更深层的问题

公司鼓励 AI 使用，但员工的 AI 使用层级**参差不齐**：
- 有些人在用豆包聊天（聊天级）
- 有些人在用 Claude Code 写代码（项目级）
- 极少数人在用 lark-cli + Agent 做工作规划（自主规划级）

公司缺少让新人从第一天就建立正确 AI 协作心智的机制。

---

## 二、你的解决方案是什么？

**"敞亮伴侣"** — 基于 lark-cli 的桌面端 AI 入职导航 Agent。

### 产品形态

- Windows/macOS 桌面应用（因依赖 lark-cli CLI，无法纯 Web 实现）
- 对话式交互界面
- 预集成 lark-cli 全权限（文档、IM、Wiki、Sheet、搜索），开箱即用

### 两大知识域

| 知识域 | 内容 | 来源 |
|--------|------|------|
| **公司通用** | 企业文化、规章制度、薪酬福利、HR/行政/财务流程、新人培训计划 | 公司规章制度 Wiki、敞亮文化 Wiki、飞书表格 |
| **部门业务** | 产品背景与演进、技术架构、当前版本需求、团队分工、技术选型、代码仓库 | 部门 Wiki、IM 群聊、技术文档、需求文档 |

### 两种交互模式

| 模式 | 行为 | 示例 |
|------|------|------|
| **Pull（主动查询）** | 用户向 Agent 提问 | "我们部门用的什么技术栈？""公司公积金比例是多少？" |
| **Push（主动推送）** | Agent 分析上下文后主动推送 | "你的培训计划显示本周要完成座席端融合方案，建议先看这 3 篇文档。" |

### 技术本质

一个预配置好飞书全上下文的桌面端 AI Agent 壳。底层通过 lark-cli 读取飞书所有数据源，上层是一个对话式交互界面。

## 三、能够带来什么价值？

| 维度 | 预期价值 |
|------|----------|
| **新人上手速度** | 从"第一周翻文档"到"第一天就有上下文"，关键信息获取时间缩短 70%+ |
| **导师时间释放** | 减少 50%+ 的重复性问答（"OpenIM 是什么""代码仓库在哪"） |
| **AI 能力基线** | 新员工入职第一天就接触 Agent 级 AI 协作，建立正确心智 |
| **信息公平** | 每个人新人都能获取同样的上下文，不依赖"问对人" |
| **跨部门复用** | 一个产品，所有事业部新人都能直接使用 |

## 五、为什么你的方案与众不同？（核心创新点）

1. **不是又一个 RAG 问答机器人**。关键区别：基于 lark-cli，能读飞书 IM、Wiki、Sheet、Doc、搜索结果全部数据源，不是只吃几篇上传的 PDF。

2. **主动推送（Push 模式）**。市面上的 AI 助手都是被动问答，敞亮伴侣通过读取新人的培训计划、群聊消息、当前版本需求，**主动建议**该关注什么。

3. **双重上下文架构**。公司通用制度 + 部门业务知识，分层组织，新人可以切换上下文域。

4. **AI 能力基线植入**。这个产品本身就是 AI Agent——新人在使用它的过程中，就在学习如何与 AI 协作。它既是信息获取工具，也是 AI 协作的入门教程。

## 六、如何验证它是否可行？

### 试点方案

- 选取智能客服产品开发组的 2 位新人（fly + Zavier）进行 A/B 对比
- 试点周期：4 周原型开发完成后，伴随下一位新人入职试用

### 验证指标

| 指标 | 目标 |
|------|------|
| 自主获取信息量 | 新人第一天可回答 80% 的常见问题 |
| 导师问答频率 | 基础问题减少 50% 以上 |
| 业务理解深度 | 第一周结束时能画出完整的产品架构图 |
| 新人满意度 | 4.5/5 分 |

### 落地规划

- 试点达标 → 推广至 LP2-HG 事业部所有新人
- 二期 → 跨事业部推广 + 知识域模块化（各部门可自定义自己的知识域）
