## 技术选型

| 层 | 方案 | 原因 |
|----|------|------|
| 桌面壳 | Electron + Vue 3 | 复用现有前端技术栈（customer-im-h5 也是 Vue 3） |
| 数据接入 | lark-cli CLI | 已有全权限集成 |
| 知识索引 | 本地向量数据库（如 LanceDB） | 无需服务端，数据本地存储 |
| AI 对话 | Claude API（通过 lark-cli 调用） | 公司 AI LAB 推荐 |
| Push 通知 | Electron Notification API | 原生系统通知 |

备选：桌面壳可考虑 Tauri（更轻量，但生态不如 Electron 成熟）。


## 两种交互模式实现

### Pull（用户主动问）

```
用户输入问题
  → Agent 判断问题属于哪个知识域
    → 调用 lark-cli 搜索相关文档
      → 获取文档内容
        → 结合上下文生成回答
          → 返回答案 + 引用来源
```

### Push（Agent 主动推）

```
定时任务 (每天早晨)
  → 读取用户培训计划 (Sheet)
    → 对比当前进度
      → 读取当前版本需求文档
        → 检索相关群聊最近讨论
          → 生成今日建议
            → 系统通知：📬 "本周你的任务是 XXX，建议今天先看这 2 篇文档"
```

## 第一阶段交付范围（4 周原型）
