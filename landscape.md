# 敞亮科技 AI 使用现状分析

> 基于入职一周的观察与群聊上下文分析。

## AI 使用的三个层级

### L1: 聊天级
- **工具**: 豆包、ChatGPT 网页版、Kimi
- **场景**: "这个报错是什么意思""帮我翻译这段话"
- **特点**: 单次问答，无上下文，无工程集成
- **局限**: 只解决"当前行"的问题，不解决"项目任务怎么收口"

### L2: 项目级
- **工具**: Claude Code、Cursor、Copilot、Trae
- **场景**: 代码生成、Code Review、Bug 修复、文件重构
- **特点**: 有工程上下文，任务驱动，能写-验证-修复闭环
- **代表**: 部分开发同学在用 Claude Code / Cursor 写代码
- **局限**: 局限在代码域，不涉及工作规划和信息获取

### L3: 自主规划级
- **工具**: lark-cli + Claude Code Agent
- **场景**: AI 主动读取飞书上下文 → 理解项目全景 → 规划行动计划 → 沉淀经验
- **特点**: AI 不只是工具，是工作搭档
- **代表**: fly（用 lark-cli + Claude 管理成长日志、扫文档、整理提案）
- **门槛**: 需要理解 Agent 协作模式，需要配置飞书权限

## 核心问题

```
【现状】AI 使用水平完全依赖个人探索
      新人入职没有 AI 使用指引
      大部分停留 L1，少数进入 L2，极个别到 L3

【后果】同样作为新人，有人第一周就能让 AI 帮他规划工作
       有人还在问豆包"帮我写个登录页面"
       差距在入职第一周就拉开了
```

## 公司 AI 文化现状

- AI LAB 有 [AI编程落地指引](https://openvision.sg.larksuite.com/wiki/Oo89dTLFKo6g1AxXqMalO9PwgNd)，明确推荐 Claude Code 作为主执行面
- 有 [SKILLS超级进化] 等文档分享 AI 技能构建方法
- "敞亮AI共创计划"首届大赛正在进行
- 公司明确鼓励 AI 应用创新

## 机会

敞亮伴侣本身就是一个 **L3 级 AI 产品**。新人使用它的过程中，自然理解：
1. AI 可以主动获取上下文（非被动问答）
2. AI 可以跨数据源整合信息（飞书全数据）
3. AI 可以做工作规划和经验沉淀

这就是"AI 能力基线植入"的含义：不是在培训里讲 AI 怎么用，而是让新人第一天就和 AI 搭档工作。
